# 文献工作流（步骤 3）

目标：为 **每一座步骤 2 缺陷** 配备至少一条可引用、可追溯的文献（课程常见要求为学术或权威行业来源；格式以课程 PDF 为准，通常 APA/HKU 体系）。

## 原则

- **一条缺陷 ↔ 至少一篇核心文献**（同一篇可覆盖相近根因时，在矩阵里说明范围，避免拼凑）。  
- 文献论点应能支撑 **修复策略或论证为何是缺陷**（无障碍、表单校验、客户端数据一致性等）。  
- 遵守学术诚信与课程 **AI 使用** 限制：直接引用须核对原文；生成式 AI 仅作辅助时需符合作业声明。

## 检索策略

1. 从 `defect-log-template.md` 中抽取 **关键词**：缺陷类型 + 场景（如 `client-side validation inventory`、`WCAG 2.2 landmark`、`DataTables accessibility`）。  
2. 优先：**ACM / IEEE Xplore / Springer / Wiley** 书籍章节、标准（W3C/WCAG）、或 **peer-reviewed** 期刊。行业白皮书需组内一致认可其可信度。  
3. 近五年优先；经典基础论文除外。  
4. 记录 **DOI / ISBN / 稳定 URL** 与 **访问日期**（网页时）。

## 与修复的对应关系

在正式报告中建议写：

- 缺陷 Di 的 **问题陈述** → 文献中的 **已有结论** → 你组 **选型中的修复**（一段话闭环）。

若文献与实现有张力（例如文献推荐服务端校验，而你只能做客户端），诚实写 **限制与仍适用的部分**。

## 矩阵与定稿

使用 `docs/templates/literature-matrix-template.md` 维护 **缺陷 ID × 文献 ID**；定稿 bib 后锁版本，与最终报告参考文献表一致。
