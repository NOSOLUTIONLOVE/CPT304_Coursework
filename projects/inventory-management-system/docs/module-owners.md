# 模块化功能域与负责人

将单页应用拆成 **协作责任域**，便于并行开发与评审。姓名/学号为占位，组会填实。

| 功能域 | 主要职责 | 主负责人 | 备份 | 主要对接工件 |
|--------|----------|----------|------|----------------|
| 壳与无障碍 | `lang`、landmark、`title`/`h*` 层级、跳过链接、焦点管理 | | | HTML 骨架、ARIA、全局 CSS |
| 表格与 CRUD | DataTables 配置、增删改模态、工具栏与导出触发 | | | `index.html` 内表格区、相关内联脚本 / `theme.js` 等 |
| 本地持久化 | `LOAD DATA` / `SAVE DATA`、`DB.txt` 格式、异常数据 | | | 文件读写与 Blob、样例 `DB.txt` |
| 样式与品牌 | Bootstrap 与 `inventory.css`、响应式与对比度 | | | CSS、图标字体加载 |
| 质量与五项基准 | 测试覆盖率、部署、Lighthouse、双语、Cookie/隐私页统筹 | | | CI、托管、清单 `professional-edition-checklist.md` |

## 并行约定

1. **改前先拉 main**，小步提交；HTML 与 JS 同时改时优先沟通「表格与 CRUD」与「壳与无障碍」接口。  
2. **持久化格式**变更须通知全组并更新样例 `DB.txt`。  
3. **无障碍与 i18n** 字符串常交织：建议由「壳与无障碍」与「质量与五项基准」双人审一遍再合并。
